# ADR-008 - Canvas Credential Storage

Date: 2026-07-05

Status: Accepted

## Context

Phase 5A uses Canvas personal access tokens generated by individual Canvas users
because they are the smallest implementation path for validating Canvas
connectivity and course discovery in the capstone slice. Stay Focused does not
have, require, or use a school-wide Canvas token.

Each personal access token belongs only to the Canvas user who generated it,
inherits only that user's Canvas permissions, and cannot bypass locked, hidden,
unpublished, or permission-restricted content. Personal access tokens are
sensitive server secrets and must not be exposed to mobile bundles, persisted on
the device, logged, or returned from the API.

`CANVAS_TOKEN_ENCRYPTION_KEY` is an application-owned encryption secret. It is
not supplied by Canvas, the school, or the student.

## Decision

Canvas personal access tokens are submitted only to the protected Stay Focused
API. The API validates the token against Canvas before persistence. Valid tokens
are encrypted with Node's built-in AES-256-GCM support using
`CANVAS_TOKEN_ENCRYPTION_KEY`, a random IV, an authentication tag, and an
encryption version.

Encrypted credentials remain server-only. Mobile does not permanently store the
Canvas token and receives only safe connection metadata. Disconnect deletes the
stored credential for that Stay Focused user.

Canvas OAuth remains the intended future production authorization path for broad
multi-user deployment, but it requires a Canvas Developer Key controlled by the
institution or administrator.

## Consequences

- API routes that encrypt or decrypt Canvas tokens require the Node runtime.
- Decryption fails closed; there is no plaintext fallback and no generated
  permanent key.
- Replacement credentials are validated before the existing connection is
  changed.
- No Canvas token may be shared across Stay Focused users.
- The developer-owned `CANVAS_PERSONAL_ACCESS_TOKEN` value is only for direct
  live validation tooling and is not a shared runtime credential for
  application users.
- Live validation requires a safely configured HTTPS API and real Canvas
  credentials; mocked automated tests are sufficient for local implementation
  when those are unavailable.
