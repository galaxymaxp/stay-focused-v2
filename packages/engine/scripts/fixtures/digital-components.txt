Digital vs. Analog - Digital • Examples • Digital Input: Push button (Pressed or not) Digital Output: LED (ON or OFF) Digital vs. Analog - Analog Analog • Analog Signals can have a range. • Between 0V – 5V • Continuous measurement is needed • Analog Pins • Analog Input: any value between 0 – 1023 using ADC (Analog- to-Digital Converter) • 0=0V and 1023 = 5V • Analog Output: • Using PWM (Pulse Width Modulation) • Rapid switches between HIGH and LOW to create an average voltage Digital vs. Analog - Analog • Examples • Analog Input: Variable resistors AKA potentiometers/pots Analog Output: LED (brightness) Digital Pins Digital Pins INPUT • Don’t need to be explicitly declared as inputs with pinMode(). • Make small demands on circuit to change from 1 state to another • Equivalent to 100megohm • However, this will report random changes in pin state. Can be from: • Noise from environment • Capacity coupling the state of nearby pin Digital Pins INPUT • Will use Pullup Resistors with pins configured as INPUT Digital Software Coding Digital Software Coding pinMode () • Configures the specified pin to behave either as an input or an output. • Syntax • pinMode(pin, mode) • Parameters • pin: the Arduino pin number to set the mode of. • mode: INPUT, OUTPUT, OR INPUT_PULLUP • Returns • Nothing • Note • The analog input pins can be used as digital pins, referred to as A0, A1, etc. Digital Software Coding pinMode () Example Code: Digital Software Coding digitalWrite () • Write a HIGH(5V or 3.3V on 3.3 board) or a LOW(0V) value to a digital pin. • Syntax • pinMode(pin, mode) • Parameters • pin: the Arduino pin number. • mode: HIGH, LOW • Returns • Nothing • Note • The analog input pins can be used as digital pins, referred to as A0, A1, etc. The exception is the Arduino Nano, Pro Mini, and Mini’s A6 and A7 pins, which can only be used as analog inputs. Digital Software Coding digitalWrite () Example Code: Digital Software Coding digitalRead () • Reads the value from a specified digital pin, either HIGH or LOW. • Syntax • digitalRead(pin) • Parameters • pin: the Arduino pin number you want to read • Returns • HIGH or LOW • Note • If the pin isn’t connected to anything, digitalRead() can return either HIGH or LOW (and this can change randomly).
